---

## Big Ideas

- Semantic HTML conveys meaning
- Forms need programmatic label associations
- Tables structure data, not layouts

Diagram:

```
<label for="name">Name:</label>
<input id="name">
  ↑ Association via for/id
```

---

## Vocabulary

| Term         | Example              |
| ------------ | -------------------- |
| Semantic     | `<article>`, `<nav>` |
| Label        | `<label for="id">`   |
| Alt text     | `<img alt="...">`    |
| Table header | `<th>`               |
| Caption      | `<caption>`          |

## Accessibility

- Every input must have an associated label
- Images need descriptive alt text
- Tables use `<th>` and `<caption>`
- Headings follow hierarchy

---

# Module 06

## Creative Web Projects

Generative Art & Interactive Visuals

---

## Module Purpose

Harness HTML5 Canvas + JavaScript to create _expressive_, _parameter-driven_ visuals demonstrating algorithmic thinking and aesthetic decisions.

---

## What You Will Learn

- Generative art concepts (rules, randomness, emergence)
- Animation loops (`requestAnimationFrame`)
- Parameterization & UI controls (sliders, buttons)
- Canvas transformations (translate, rotate, save/restore)
- Particle systems & pattern algorithms
- Iterative creative process documentation

---

## Why Generative Art?

Blends STEAM domains:

- **Science/Math:** Patterns, trigonometry, ratios
- **Technology:** Canvas API, event handling
- **Art/Design:** Composition, palette, rhythm
- **Engineering:** Performance & optimization

---

## Module Structure

Single Unit: **6.1 Generative Art**

1. Foundations: Canvas + Animation
2. Random & Parametric Patterns
3. Transformations & Particle Systems
4. UI Integration & Control Exposure
5. Iteration, Refinement, Reflection

---

## Learning Outcomes

By completion students can:

- Build animated code-based artworks
- Expose parameters for interactive exploration
- Optimize performance (object reuse, efficient loops)
- Reflect critically on aesthetics & algorithmic choices
- Prepare for purpose-driven app creation (Module 07)

---

## Key Concepts

| Concept    | Description                        |
| ---------- | ---------------------------------- |
| Algorithm  | Steps defining visual generation   |

- Invite culturally diverse pattern inspirations

## Iterative Process

1. Concept sketch
2. Minimal prototype
3. Parameter exposure
4. Palette refinement
5. Performance tuning
6. Documentation & reflection

---

## Common Pitfalls & Fixes

| Pitfall                | Fix                             |
| ---------------------- | ------------------------------- |
| Degenerate performance | Reuse objects; reduce count     |
| Visual chaos           | Constrain random ranges         |
| Lost rotation control  | Use save/restore, center origin |
| Unresponsive UI        | Use `input` event, parse values |
